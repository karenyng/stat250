Why interpretted languages is slower than compiled languages:

boxing - the interpretter has to check the syntax and find what the syntax means , e.g. what "=" does etc. before it can hand the instructions to the CPU for computation also the syntax finding is done for every variable / symbol

compilation bind all the syntax into one big filer so the syntax finding part is much faster, also the syntax searching is done only once
